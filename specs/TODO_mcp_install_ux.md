## Research Summary

### Cursor
- Deep link format: `cursor://anysphere.cursor-deeplink/mcp/install?name=NAME&config=BASE64_CONFIG`
- **Critical**: Cursor's native HTTP/SSE remote MCP transport is **broken in v3.0.9+** (12+ month regression, unfixed as of April 2026). Affects Figma, Vercel, and all remote HTTP MCPs.
- **Solution**: Use `mcp-remote` (npm) as a stdio bridge. Config must be:
  ```json
  {"type": "stdio", "command": "npx", "args": ["-y", "mcp-remote", "https://your-docs.com/mcp"]}
  ```
- Cursor Marketplace exists at [cursor.com/marketplace](https://cursor.com/marketplace) — plugins can be submitted there.

### VS Code
- Deep link: `vscode://mcp/install?URL_ENCODED_CONFIG` — supports HTTP transport natively.
- Config: `{"type": "http", "url": "https://your-docs.com/mcp"}`

### Claude Desktop
- **No URL scheme** for remote HTTP MCP servers.
- `.mcpb` extensions (Settings > Extensions) are for local, bundled stdio servers only — not applicable to ServeM's remote HTTP endpoint.
- Remote HTTP support: manually edit `claude_desktop_config.json` (`Settings > Developer > Edit Config`).
- Best UX: show config snippet with a **copy-to-clipboard** button (same pattern as existing code-block copy).
- Config path: macOS `~/Library/Application Support/Claude/claude_desktop_config.json`, Windows `%APPDATA%\Claude\claude_desktop_config.json`.

### UI Placement
The **"Powered by ServeM" sidebar footer** is the entry point — it links to a dedicated internal **`/servemd`** page that renders in the full three-column layout (sidebar + content + TOC), just like any other docs page. The page is dynamically generated by FastAPI (no `.md` source file), shows version info, install buttons, and setup instructions, and is **excluded from the Whoosh search index** since it's not under `DOCS_ROOT`.

Benefits over an inline expandable panel:
- Full content area — room for proper install instructions per client, not just buttons
- Linkable and shareable URL
- TOC navigation for the sections
- Reuses the existing `create_html_template()` infrastructure unchanged

## Appendix: Deep Link Formats (Verified April 2026)

### Cursor (via mcp-remote — required workaround for broken HTTP transport)
```
cursor://anysphere.cursor-deeplink/mcp/install?name=My+Docs&config=BASE64
```
Where `config` decodes to:
```json
{"type": "stdio", "command": "npx", "args": ["-y", "mcp-remote", "https://your-docs.com/mcp"]}
```
Why `mcp-remote`: Cursor v3.0.9+ has a regression where native HTTP MCP transport fails to fall back from Streamable HTTP to SSE, causing all remote HTTP MCPs to break. `mcp-remote` bridges via stdio and is the officially recommended community workaround.

### VS Code (native HTTP — works without workaround)
```
vscode://mcp/install?%7B%22name%22%3A%22docs%22%2C%22type%22%3A%22http%22%2C%22url%22%3A%22https%3A%2F%2Fyour-docs.com%2Fmcp%22%7D
```
Decoded: `{"name": "docs", "type": "http", "url": "https://your-docs.com/mcp"}`

### Claude Desktop (copy-paste only — no URL scheme for remote HTTP)
```json
{
  "mcpServers": {
    "your-docs": {
      "type": "http",
      "url": "https://your-docs.com/mcp"
    }
  }
}
```
File location:
- macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows: `%APPDATA%\Claude\claude_desktop_config.json`

### Python link generators (reference implementation)
```python
import base64, json, urllib.parse

def cursor_install_link(name: str, mcp_url: str) -> str:
    """Cursor deep link using mcp-remote (workaround for broken HTTP transport)."""
    config = json.dumps({"type": "stdio", "command": "npx", "args": ["-y", "mcp-remote", mcp_url]})
    b64 = base64.b64encode(config.encode()).decode()
    return f"cursor://anysphere.cursor-deeplink/mcp/install?name={urllib.parse.quote(name)}&config={b64}"

def vscode_install_link(name: str, mcp_url: str) -> str:
    """VS Code deep link — native HTTP transport."""
    config = json.dumps({"name": name, "type": "http", "url": mcp_url})
    return f"vscode://mcp/install?{urllib.parse.quote(config)}"
```
